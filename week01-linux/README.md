## Day 01 – 리눅스 기본 명령어 (기준선 확인)

이번 Day 1은 새로운 내용을 학습하기보다는  
서버 운영 관점에서  
리눅스 기본 명령어에 대한 나의 기준선을 확인하는 목적이었다.

### 확인한 명령어
- pwd, ls, cd
- mkdir, touch, rm
- cat, tail

### 서버 관점에서 중요하다고 느낀 점
- 현재 작업 디렉터리를 항상 먼저 확인해야 한다
- rm 명령어는 실서버에서 치명적인 사고로 이어질 수 있다
- 장애 발생 시 코드보다 로그를 먼저 확인해야 한다

### 느낀 점
이미 알고 있던 내용이었지만  
개발 관점이 아닌 운영 관점으로 다시 보니  
명령어 하나하나에 대한 긴장도가 달라졌다.
