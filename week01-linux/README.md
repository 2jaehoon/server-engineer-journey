# Week 01 – Linux Fundamentals

서버/플랫폼 엔지니어 관점에서  
리눅스 기본 명령어와 운영 감각을 다시 점검하는 주차이다.

## Day 01 – 리눅스 기본 명령어 (기준선 확인)

이번 Day 1은 새로운 내용을 학습하기보다는  
서버 운영 관점에서  
리눅스 기본 명령어에 대한 나의 기준선을 확인하는 목적이었다.

### 확인한 명령어
- pwd, ls, cd
- mkdir, touch, rm
- cat, tail

### 서버 관점에서 중요하다고 느낀 점
- 현재 작업 디렉터리를 항상 먼저 확인해야 한다
- rm 명령어는 실서버에서 치명적인 사고로 이어질 수 있다
- 장애 발생 시 코드보다 로그를 먼저 확인해야 한다

### 느낀 점
이미 알고 있던 내용이었지만  
개발 관점이 아닌 운영 관점으로 다시 보니  
명령어 하나하나에 대한 긴장도가 달라졌다.

## Day 02 – 사용자와 권한

서버에서 발생하는 대부분의 문제는  
"권한"과 "사용자" 개념을 이해하면 설명할 수 있다.

### 실습 내용
- 파일 권한 확인 및 변경 (ls -l, chmod)
- 권한이 없는 상태에서의 접근 시도
- sudo를 통한 관리자 권한 실행

### 서버 관점에서 느낀 점
- Permission denied는 에러가 아니라 보호 장치였다
- sudo는 모든 작업에 쓰는 도구가 아니라,
  꼭 필요할 때만 사용해야 하는 권한 상승 수단이다

### 추가로 떠오른 질문
- 왜 파일 권한을 소유자 / 그룹 / 기타 사용자로 나눴을까?
- 왜 실행 권한(x)을 따로 두었을까?
- sudo를 쓴 기록은 어디에 남을까?
