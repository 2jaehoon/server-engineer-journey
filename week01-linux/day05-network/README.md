# Day 5 – Network Basics (Port & Socket)

Day 5에서는 서버가 “열려 있다”는 말의 정확한 의미를
프로세스와 네트워크 관점에서 이해하는 데 집중했다.

단순히 포트 번호를 외우는 것이 아니라,
실제로 어떤 프로세스가 어떤 상태로
네트워크 요청을 받고 있는지를 확인하는 흐름을 정리했다.

---

## 학습 내용

### 포트와 프로세스의 관계
- 포트는 단독으로 존재하지 않고, 항상 프로세스가 점유한다
- 서버가 떠 있다는 것은 특정 프로세스가 포트를 LISTEN 상태로
  소켓에 바인딩하고 있다는 의미다

### ss 명령어의 역할
- `ss`는 커널이 관리하는 네트워크 소켓 상태를 조회하는 도구다
- 어떤 프로세스가 어떤 포트를 LISTEN 하고 있는지 확인할 수 있다
- 단순 실행 명령이 아니라 상태 확인을 위한 진단 도구라는 점이 중요하다

### 0.0.0.0과 127.0.0.1의 차이
- `127.0.0.1`은 로컬에서만 접근 가능한 주소다
- `0.0.0.0`은 모든 네트워크 인터페이스에서
  연결을 받을 수 있도록 바인딩한 상태를 의미한다
- 외부 접속이 되지 않는 문제의 원인으로
  주소 바인딩 설정을 반드시 확인해야 한다

---

## 접속 장애 점검 흐름

서버 접속이 되지 않을 때는
네트워크보다 서버 내부 상태부터 확인하는 것이 중요하다.

1. `ps -ef`로 프로세스가 실행 중인지 확인한다
2. PID를 기준으로 `ss -lntp`를 통해
   해당 프로세스가 포트를 LISTEN 하고 있는지 확인한다
3. LISTEN 상태가 존재한다면,
   `ss -ntp`를 통해 실제 연결(ESTABLISHED)이 발생하는지 확인한다

LISTEN 상태는 요청을 받을 준비 상태이고,
ESTABLISHED 상태는 실제 요청이 서버에 도달했다는 증거다.

---

## 느낀 점

- 서버 장애의 원인은 네트워크가 아니라
  서버 내부 상태에서 해결되는 경우가 많다는 것을 느꼈다
- 이전에는 막연하게 “포트가 열려 있다”고 생각했지만,
  이제는 프로세스와 소켓 상태로 설명할 수 있게 되었다

---

## 한 줄 정리

서버가 열린다는 것은,
특정 프로세스가 특정 주소와 포트에서
소켓을 LISTEN 상태로 점유하고 있다는 의미다.
