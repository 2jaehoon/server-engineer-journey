## Weekend Mini Project – Service Control Script

리눅스 환경에서 하나의 애플리케이션을
**서비스처럼 실행 / 종료 / 상태 확인**할 수 있도록
간단한 제어 스크립트를 직접 구현해보는 미니 프로젝트입니다.

### 목적
- 단순 실행이 아닌 **운영 관점에서의 서비스 관리 흐름 이해**
- PID 파일 기반 프로세스 관리 방식 체험
- “실행 중인가?”를 파일이 아닌 **실제 프로세스 기준으로 판단**하는 경험

---

### 구성 파일
- `app.sh`  
  무한 루프로 동작하며 로그를 출력하는 테스트용 애플리케이션
- `svc.sh`  
  서비스 제어 스크립트 (start / stop / status / logs)
- `logs/`  
  애플리케이션 로그 디렉터리
- `run/`  
  PID 파일 저장 디렉터리

---

### 구현한 기능
- `nohup` + 백그라운드 실행으로 터미널과 분리된 서비스 실행
- PID 파일(`run/app.pid`)을 통한 실행 상태 추적
- `kill -0`을 사용해 PID 파일을 맹신하지 않고 **실제 프로세스 존재 여부 확인**
- 프로세스는 종료되었지만 PID 파일만 남아있는  
  **stale pidfile 상황 재현 및 처리**
- 실무에서 자주 사용하는 start / stop / status / logs 인터페이스 구현

---

### 배운 점
- PID 파일은 “사실”이 아니라 **상태에 대한 주장**이라는 점
- 서비스 운영에서 가장 중요한 것은 실행 자체보다  
  **정확한 상태 판단과 복구 가능성**이라는 점
- 단순한 스크립트라도 운영 관점으로 보면 고려할 요소가 많다는 것을 체감함
