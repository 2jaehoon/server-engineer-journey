# Day 17 – Database Failure Observation

이번 실습에서는 DB 컨테이너를 중지시켜
의존 서비스 장애 상황을 인위적으로 만들고,
애플리케이션의 실제 동작을 관찰했다.

---

## 실습 내용
- 실행 중인 DB 컨테이너를 중지(stop)
- app 컨테이너 상태 및 로그 확인

---

## 관찰 결과
- DB가 중지되었음에도 app 컨테이너는 종료되지 않고 실행 상태를 유지했다
- `depends_on: condition: service_healthy` 설정은
  컨테이너 시작 순서에만 영향을 주며,
  실행 중 DB 상태 변화에는 영향을 주지 않는다
- 따라서 컨테이너 실행 여부만으로 서비스 상태를 판단하기 어렵다

---

## 느낀 점
- 실행 중인 프로세스가 존재한다고 해서
  서비스가 정상 동작한다고 볼 수는 없다
- 의존 서비스 장애 시
  애플리케이션이 어떤 방식으로 실패하는지를 확인하는 것이 중요하다
- 장애 상황은 단순히 “죽는다/안 죽는다”보다
  “어떤 형태로 드러나는가”를 중심으로 봐야 한다
