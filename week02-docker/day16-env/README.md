# Day 16 – Environment Variables (.env)

Day 16에서는 docker-compose 설정에서  
환경 변수를 코드에 직접 작성하지 않고  
`.env` 파일로 분리하는 방법을 학습했다.

이는 보안과 운영 안정성 측면에서  
서버 및 컨테이너 환경에서 반드시 필요한 구성 방식이다.

---

## 왜 환경 변수를 분리해야 할까?

- 비밀번호, 포트, 계정 정보 등을 코드에 하드코딩하면 위험하다
- 개발 / 테스트 / 운영 환경마다 설정이 달라질 수 있다
- GitHub에 민감 정보가 노출될 가능성을 차단해야 한다

---

## 적용한 내용

- `.env` 파일을 사용해 DB 계정, 포트 정보를 분리
- `docker-compose.yml`에서 `${VARIABLE}` 형태로 참조
- `.gitignore`에 `.env`를 추가하여 Git 추적에서 제외
