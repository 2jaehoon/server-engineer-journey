# Day 10 – Dockerfile로 실행 환경 정의하기

Day 10에서는 docker run 명령어로 컨테이너를 실행하는 방식에서 나아가,
Dockerfile을 사용해 실행 환경 자체를 코드로 정의하는 방법을 학습했다.

이번 학습의 핵심은
컨테이너 실행을 “명령어 실행”이 아니라
“재현 가능한 환경 정의”로 전환하는 것이었다.

---

## docker run 방식의 한계

docker run 명령어는 컨테이너를 빠르게 실행할 수 있지만,
다음과 같은 한계가 있다.

- 실행 옵션을 매번 기억해야 한다
- 실행 방식이 문서로 남지 않는다
- 동일한 환경을 반복해서 재현하기 어렵다

이러한 문제는
서버 환경이나 협업 상황에서 관리 포인트가 된다.

---

## Dockerfile의 역할

Dockerfile은
컨테이너가 어떻게 만들어지고 실행되어야 하는지를
코드로 정의하는 파일이다.

Dockerfile을 사용하면 다음이 가능해진다.

- 실행 환경을 명확하게 기록할 수 있다
- 동일한 환경을 언제든지 재현할 수 있다
- 컨테이너 실행 방식을 표준화할 수 있다

즉, Dockerfile은
컨테이너 실행 환경의 설계도 역할을 한다.
