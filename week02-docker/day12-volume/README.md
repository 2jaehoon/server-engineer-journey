# Day 12 – Docker Volume

Day 12에서는 Docker 컨테이너의 휘발성과
데이터 영속성 문제를 이해하기 위해 Volume을 학습했다.

---

## 컨테이너와 데이터의 관계

Docker 컨테이너는 실행 환경을 제공하는 단위로,
컨테이너가 종료되거나 삭제되면
컨테이너 내부에 생성된 파일도 함께 사라진다.

이는 Docker의 설계 철학에 따른 정상적인 동작이다.

---

## Volume의 역할

Volume은 컨테이너와 분리된 데이터 저장 공간으로,
컨테이너가 삭제되더라도 데이터를 유지할 수 있다.

컨테이너는 언제든 재생성될 수 있기 때문에,
로그, 설정 파일, 서비스 데이터와 같은 정보는
Volume을 통해 관리해야 한다.

---

## Volume 실습

nginx 컨테이너 실행 시 Volume을 마운트하여
컨테이너 내부에서 생성한 파일이
컨테이너 재생성 이후에도 유지되는 것을 확인했다.

이를 통해 실행 환경과 데이터는
명확히 분리해서 관리해야 한다는 점을 이해했다.

---

## 정리

컨테이너는 휘발성 실행 환경이고,
데이터는 Volume을 통해 영속적으로 관리해야 한다.

이는 Docker 기반 서비스 운영에서
가장 기본적이고 중요한 설계 원칙이다.
