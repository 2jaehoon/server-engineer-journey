# Day 09 – Docker 설치 및 첫 컨테이너 실행

Day 9에서는 Docker를 설치하고,
첫 컨테이너를 직접 실행해 보며
Docker 컨테이너가 무엇인지 직접 확인했다.

이번 실습의 목표는
Docker 컨테이너를 가상 서버가 아닌
“격리된 프로세스 실행 방식”으로 이해하는 것이었다.

---

## Docker 실행 환경 확인

WSL(Ubuntu) 환경에서 Docker를 설치하고,
Docker 데몬이 정상적으로 실행 중인지 확인했다.

Docker는 리눅스 커널 위에서 동작하며,
컨테이너는 결국 하나의 프로세스로 실행된다.

---

## nginx 컨테이너 실행

다음 명령어로 nginx 컨테이너를 실행했다.

```bash
docker run -d -p 8080:80 nginx
이 명령어의 의미는 다음과 같다.

nginx 웹 서버 프로세스를 컨테이너 내부에서 실행한다

컨테이너는 내부적으로 80 포트를 LISTEN 한다

호스트의 8080 포트를 컨테이너의 80 포트와 매핑한다

-d 옵션을 통해 백그라운드에서 실행한다

컨테이너 입장에서는
자신이 8080 포트를 사용하고 있다는 개념이 없으며,
단순히 80 포트에서 서비스하고 있을 뿐이다.
