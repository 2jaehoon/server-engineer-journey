# Day 18 – Detecting Failure via Logs

이번 실습에서는 DB 장애 발생 시
애플리케이션 로그에 어떤 변화가 나타나는지 확인했다.

---

## 실습 내용
- 정상 상태에서 app 로그 확인
- DB 컨테이너 중지 후 app 로그 변화 관찰

---

## 관찰 결과
- DB 중지 이후 app 로그에
  DB 연결 실패 관련 에러 메시지가 기록되었다
- 컨테이너 상태는 유지되었지만,
  로그를 통해 실제 기능 장애를 확인할 수 있었다

---

## 정리
- 컨테이너 실행 상태만으로는 장애 여부를 판단하기 어렵다
- 로그는 장애를 가장 먼저 드러내는 신호 중 하나다
- 장애 판단은 상태(status)가 아니라 증상(symptom)을 기준으로 해야 한다
